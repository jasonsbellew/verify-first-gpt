# Verify First Output Templates

| Field | Value |
| --- | --- |
| File role | User-facing response templates |
| Runtime placement | Upload as public GPT knowledge |
| Version | 0.1.0 |
| Updated | 2026-07-05 |

## Quick verdict

Use for short checks.

```text
Verdict: [Verified / Mostly true / Mixed / Unclear / Unsupported / Likely false / False]
Confidence: [High / Medium / Low]

Short answer:
[1-3 sentences]

Key caveat:
[Most important limitation, date issue, source issue, or context issue]
```

## Claim breakdown

Use for multi-part claims.

```text
Overall verdict: [label]
Confidence: [level]

Claims checked:
1. [Subclaim] - [verdict] - [why]
2. [Subclaim] - [verdict] - [why]
3. [Subclaim] - [verdict] - [why]

Bottom line:
[Plain-language conclusion]

What would change this:
[Evidence needed]
```

## Source table

Use when multiple sources matter.

```text
| Source | Type | Date | Supports? | Notes |
| --- | --- | --- | --- | --- |
| [Name/link] | [Primary/secondary/etc.] | [Date] | [Yes/No/Partial] | [Limitations] |
```

## Source audit

Use when evaluating an article, post, chart, or document.

```text
Source reliability for this use: [High / Medium / Low]

What the source claims:
- [Claim]
- [Claim]

What it actually supports:
- [Supported point]

Gaps or concerns:
- [Concern]

Best next source:
[Primary source or stronger evidence to inspect]
```

## Decision brief

Use when the user needs to act.

```text
Decision context:
[What the user is deciding]

Verified facts:
- [Fact]

Assumptions:
- [Assumption]

Unknowns:
- [Unknown]

Risk if wrong:
[Consequence]

Safer next step:
[Verification step]
```

## Better wording

Use when a claim is almost right but needs correction.

```text
Instead of:
"[Original claim]"

Say:
"[More accurate wording]"

Why:
[Brief explanation]
```
