# Verify First Verification Workflows

| Field | Value |
| --- | --- |
| File role | Reusable verification workflows |
| Runtime placement | Upload as public GPT knowledge |
| Version | 0.1.0 |
| Updated | 2026-07-05 |

## Fast claim check

Use for low-risk claims or when the user wants a quick answer.

1. Restate the claim.
2. Decide whether it is time-sensitive.
3. Check the best available source or explain what source is needed.
4. Give a verdict and confidence.
5. Add one caveat.

## Deep verification

Use for disputed, high-stakes, complex, or publication-ready claims.

1. Break the claim into subclaims.
2. Classify each subclaim.
3. Identify the strongest expected evidence.
4. Inspect primary sources first when available.
5. Compare independent secondary sources.
6. Note source dates, methods, and incentives.
7. Separate established facts from inference.
8. Give a verdict for each subclaim.
9. Give an overall verdict.
10. Explain what evidence would change the answer.

## Source audit

Use when the user provides an article, post, screenshot, video, chart, or source list.

1. Identify the source creator, date, and format.
2. Determine whether the source is primary, secondary, tertiary, or unclear.
3. Extract the main claims.
4. Check whether claims are sourced.
5. Follow citations to original evidence when possible.
6. Flag missing context, method gaps, unsupported leaps, or stale information.
7. Rate source reliability for the user's purpose.

## Quote verification

Use when a user asks whether someone said something.

1. Ask for or search for the original transcript, recording, post, document, or archived source.
2. Distinguish exact quote from paraphrase.
3. Check date and context.
4. Avoid relying on quote images or reposts unless no better evidence exists.
5. If only secondary references exist, label the claim as unverified or low confidence.

## Time-sensitive verification

Use for claims that may change.

1. Identify the current date and relevant effective date.
2. Search official or primary sources first when tools are available.
3. Prefer sources with clear publication or update dates.
4. State "as of" date when giving the answer.
5. Avoid stale recommendations when newer data may exist.

## Conflicting evidence workflow

Use when reliable sources disagree.

1. Identify exactly what each source says.
2. Compare dates, definitions, methods, scope, and incentives.
3. Prefer newer sources only when the newer source is authoritative and relevant.
4. Explain the disagreement without forcing a false tie or false certainty.
5. Give the most defensible provisional conclusion.

## Decision check

Use when the user asks whether to buy, trust, publish, hire, invest, share, or act.

1. Identify the decision and stakes.
2. List verified facts.
3. List assumptions.
4. List unknowns.
5. Identify the biggest risk of being wrong.
6. Recommend the next verification step.
7. Give a cautious decision frame if evidence is incomplete.

## No-live-source fallback

Use when current tools or source access are unavailable.

1. State that live verification is unavailable.
2. Use only information provided by the user or stable background knowledge.
3. Avoid current claims unless clearly qualified.
4. Tell the user which source would settle the question.
5. Offer a checklist the user can use outside the chat.
